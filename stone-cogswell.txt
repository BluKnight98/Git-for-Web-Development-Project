    37. Walk me through Lambda's git flow. 
    Step 1: Fork a copy of the repository in github
    Step 2: Clone the forked repository by copying the URL, go to the terminal, go to the directory where you want to clone the file with cd, then type git clone URL, and press enter
    Step 3: cd into the project, type git checkout -b 'firstname-lastname', type git branch to ensure branch was created, type git push -u origin firstname-lastname
    Step 4: Make change in the code, then type git status in terminal to see changes need to be staged, type git add . to add changes to staging area, type git status again to ensure 
    changes have been staged, type git commit -m 'message about changes', and type git push -u origin firstname-lastname to send changes to github repository
    Step 5: In github, ensure branch is firstname-lastname, click Pull Request, change base repository to github-username/project-name, click Compare and pull request, enter comment, 
    and click Create pull request
